Схема базы eco.db
===

Общая информация: база sqlite3
Предназначена для хранения информации о мониторингах, данных с мониторингов, геоданные, метеоданные

Таблицы:

1) monitor_data
(Временная) Предназначена для хранения данных с мониторингов
id
load_ts
monitoring_id
value_co

2) monitor
Предназначена для хранения основной информации о мониторингах
id              Номер мониторинга, primary key
name            Название мониторинга
url             URL для дневных данных мониторинга
url_type        Тип поставщика данных
spatial_type    Тип станции для данного поставщика (station, mobile)
is_co           Производятся ли на ней замеры CO
--is_available    Активна ли станция в данный момент

3) monitor_geo
(К сожалению в sqlite3 нет геоиндекса, так что можно брать только в квадратной области)
Предназначена для хранения гео данных о мониторингах
monitor_id      Foreign key, номер мониторинга
lon             Долгота
lat             Широта

4) monitor_data_co
Предназначена для хранения данных об CO на каждой станции
monitor_id      Foreign key, номер мониторинга
dump_ts         Время забора CO
dump_value      Значение CO в dump_ts
